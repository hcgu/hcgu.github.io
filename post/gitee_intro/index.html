Gitee最简教程（不用git版）

2022年11月11日
| 工具
| 阅读

欢迎来到 东南大学-顾洪成 的 工作台，这份Gitee最简教程专为零基础用户快速上手准备，请遵照以下步骤操作。

  1. 点击工作台 链接，并将它添加到浏览器的书签栏。

  2. 按下图设置 工作台 的组件，仅保留 我的仓库 、我的工作项 、个人数据概览 和 我的周报 共4个组件。 工作台 左侧为导航栏， 我的仓库 是 代码 的子集，后者还包括高校开源及内源仓库。请关注右侧顶端的通知，也可关注微信公众号获得实时推送。

  3. 左侧导航栏 代码 中收集了所有仓库（代码库），包括私有、内源（内部开源）和开源三类。下图中我参与的 Demo 为私有仓库，获授权用户可访问和操作。

  4. 对仓库（代码库）的操作包括三类：

    • 文件读写，包括：新建、修改、删除；
    • Web IDE，一般用于在线撰写Markdown文档，或紧急修改代码；
    • 工作项，包括：需求、任务、缺陷。 先讲第一类操作，即文件读写。可以通过 文件 下拉菜单直接新建文件（Markdown或代码）或文件夹，甚至上传已有文件。

也可右击文件进行重命名或删除操作。

  1. 右击Markdown或代码文件，选择 新标签页打开 ，或者在顶端点击 WebIDE 均启动Web IDE编辑功能。Web IDE一般用于在线撰写Markdown文档，或紧急修改代码，具体包括以下四个步骤：

    • 撰写文档或代码；
    • 点左上 暂存的更改；【该操作对应git命令：git add . 】
    • 对更新进行备注，并点左中 提交信息；【该操作对应git命令：git commit -m "msg" ，其中 msg 自定义】
    • 点左下 提交；【该操作对应git命令：git push -u origin master 】

  2. 第4步中对仓库（代码库）的操作还包括 工作项 ，分为需求、任务、缺陷这3种。需求 指工作中缺少的条件，如仿真计算遇到硬件瓶颈需升级内存、实验需购买某试剂等；任务 是分解出来的阶段工作，一般带有指标；缺陷 是回溯工作时发现待改进的、甚至错误的，如汇报代码bug、发现公式错误等。

事实上，只需获得仓库地址（而不登陆 工作台）即可完成上述所有操作，例如下图通过 Issue 也能发起 工作项。但这种 Issue 模式不要求负责人、截止时间，甚至主体部分空白都能提交成功。这种 Issue 模式过于松散，甚至无法更新到 工作台，从而影响工作效率。因此我们约定：慎用Issue模式。
